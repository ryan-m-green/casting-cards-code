📚 Cast Library Application Summary

Overview

The Cast Library is a comprehensive D&D campaign management tool that enables Dungeon Masters (DMs) to create, manage, and share campaign content including characters (casts), locations, sublocations, and player cards. The application features real-time collaboration through SignalR, robust role-based access control, and a rich Angular-based UI.

🏗️ Architecture Overview

Backend Structure

The backend follows a layered architecture pattern with clear separation of concerns:

  • CastLibrary.WebHost - ASP.NET Core API controllers and SignalR hubs
  • CastLibrary.Logic - Business logic, command handlers, and query handlers (CQRS pattern)
  • CastLibrary.Repository - Data access layer with Entity Framework Core
  • CastLibrary.Shared - Shared domain objects, entities, requests, responses, and enums
  • CastLibrary.Adapter - External service integrations
  • CastLibrary.Initializer - Database initialization and seeding

Frontend Structure

The Angular application is organized by feature modules with lazy loading for optimal performance:

  • core - Authentication, authorization guards, and hub services
  • features - Feature-specific components (campaigns, casts, locations, etc.)
  • shared - Reusable components and services
  • layout - Shell components and navigation

🔌 API Endpoints

AuthController (Public API - No Authorization Required)

Route Prefix: api/auth

  • POST /login
    Authenticates a user with email and password. Returns JWT token and user information. Rate limited to prevent brute force attacks.
  • POST /register
    Registers a new user account. Validates email format and password strength. Requires admin invite code.
  • POST /forgot-password
    Initiates password reset flow. Sends reset token via email to user.
  • POST /reset-password
    Completes password reset with new password and reset token from email.
  • POST /change-password
    Changes password for authenticated user. Requires current password verification.

CastsController (Authorized - DM Only)

Route Prefix: api/cast

  • GET /
    Retrieves all cast characters owned by the current DM user. Returns list of cast summaries with metadata.
  • GET /{id}
    Retrieves detailed information for a specific cast character by ID.
  • POST /
    Creates a new cast character with provided details (name, race, role, voice notes, etc.). Returns created cast with generated ID.
  • PUT /{id}
    Updates an existing cast character's properties. All fields are optional for partial updates.
  • DELETE /{id}
    Deletes a cast character. Returns 204 No Content on success.
  • POST /{id}/image
    Uploads a profile image for a cast character. Accepts JPEG, PNG, WebP up to 5MB. Returns new image URL.

LocationsController (Authorized - DM Only)

Route Prefix: api/locations

  • GET /
    Retrieves all locations owned by the current user. Used for location library browsing.
  • GET /{id}
    Retrieves detailed information for a specific location including description and metadata.
  • POST /
    Creates a new location with name and description. Location becomes part of user's reusable library.
  • PUT /{id}
    Updates an existing location's information.
  • DELETE /{id}
    Deletes a location from the library. May fail if location is in use by active campaigns.
  • POST /{id}/image
    Uploads a background image for a location. Accepts JPEG, PNG, WebP up to 5MB.

SubLocationsController (Authorized - DM Only)

Route Prefix: api/sublocations

  • GET /
    Retrieves all sublocations (nested locations with shops) owned by current user.
  • GET /{id}
    Retrieves detailed sublocation including shop items and description.
  • POST /
    Creates a new sublocation. Name is required. Can include shop items.
  • PUT /{id}
    Updates an existing sublocation's name, description, and shop items.
  • DELETE /{id}
    Deletes a sublocation.
  • POST /{id}/image
    Uploads an image for a sublocation.

CampaignsController (Authorized - DM & Player)

Route Prefix: api/campaigns

  • GET /
    Retrieves campaigns. Returns DM campaigns if user is DM, player campaigns if user is player. Lists basic campaign info without full details.
  • GET /{id}
    Retrieves complete campaign details including locations, casts, sublocations, secrets, relationships, and players. DM-only view.
  • GET /{id}/player
    Retrieves player view of campaign with limited visibility (no DM-only secrets, limited cast details).
  • POST /
    Creates a new campaign with name, fantasy type, and optional description. DM becomes owner.
  • PATCH /{id}
    Updates campaign metadata (name, description, status, etc.).
  • DELETE /{id}
    Deletes an entire campaign and all associated data.
  • POST /{id}/locations
    Adds a location from library to campaign. Creates a campaign-specific instance of the location.
  • PATCH /{id}/locations/{instanceId}
    Updates location instance properties specific to this campaign.
  • PATCH /{id}/locations/{instanceId}/visibility
    Toggles visibility of location to players. Sends real-time notification via SignalR.
  • POST /{id}/sublocations
    Adds a sublocation instance to campaign.
  • PATCH /{id}/sublocations/{instanceId}
    Updates sublocation instance details in campaign.
  • PATCH /{id}/sublocations/{instanceId}/visibility
    Toggles sublocation visibility to players.

CastsController - Campaign Instances (Authorized)

Route Prefix: api/campaigns/{campaignId}

  • POST /cast/{castId}
    Adds a cast to a campaign, creating a campaign-specific instance with visibility controls.
  • PATCH /cast/{castInstanceId}
    Updates a cast instance's campaign-specific properties (custom items, notes, etc.).
  • PATCH /cast/{castInstanceId}/visibility
    Controls which players can see a cast instance. Broadcasts change via SignalR.
  • DELETE /cast/{castInstanceId}
    Removes a cast instance from campaign.

CastRelationshipsController (Authorized)

Route Prefix: api/campaigns/{campaignId}/cast-relationships

  • GET /
    Retrieves all cast relationships in campaign. Can filter by source cast instance using query parameter.
  • GET /{id}
    Retrieves specific relationship details between two casts.
  • POST /
    Creates a new relationship between two cast instances (friend, rival, mentor, etc.).
  • PUT /{id}
    Updates an existing relationship type and description.
  • DELETE /{id}
    Deletes a relationship between two casts.

PlayerCardsController (Authorized)

Route Prefix: api/campaigns/{campaignId}/player-cards

  • GET /mine
    Retrieves the current player's own character card in this campaign.
  • GET /
    Retrieves all player cards in campaign. DM can see all, players can see discovered party.
  • GET /party
    Retrieves discovered party members that players have encountered.
  • POST /
    Creates a new player card for current user in campaign.
  • PUT /{playerCardId}
    Updates player card properties (name, description, etc.).
  • POST /{playerCardId}/image
    Uploads player character portrait image.
  • GET /{playerCardId}/memories
    Retrieves all memories/story notes for a player card.
  • POST /{playerCardId}/memories
    Adds a memory entry to player card with optional memory type.
  • DELETE /{playerCardId}/memories/{memoryId}
    Deletes a memory from player card.
  • GET /{playerCardId}/traits
    Retrieves all traits/abilities for a player card.
  • PUT /{playerCardId}/traits/{traitId}
    Updates a trait entry or creates if not exists (upsert).
  • DELETE /{playerCardId}/traits/{traitId}
    Deletes a trait from player card.
  • GET /{playerCardId}/secrets
    Retrieves secrets that have been revealed to player card.
  • POST /{playerCardId}/conditions
    Assigns a condition/status to player card. DM-only. Broadcasts via SignalR.
  • DELETE /{playerCardId}/conditions/{conditionId}
    Removes a condition from player card. Broadcasts via SignalR.
  • GET /{playerCardId}/perceptions
    Retrieves cast perceptions (what player thinks about specific casts).
  • PUT /{playerCardId}/perceptions/{castInstanceId}
    Updates or creates player's perception of a cast instance.

CastPlayerNotesController (Authorized)

Route Prefix: api/campaigns/{campaignId}/cast-player-notes

  • GET /{castInstanceId}
    Retrieves DM and player notes for a specific cast instance in campaign.
  • GET /by-cast-instances
    Retrieves notes for multiple cast instances at once using query parameters.
  • PUT /{castInstanceId}
    Updates or creates notes for a cast instance. Separate DM and player note fields.

LocationPoliticalNotesController (Authorized)

Route Prefix: api/campaigns/{campaignId}/Location-political-notes

  • GET /{LocationInstanceId}
    Retrieves political notes for a location instance.
  • PUT /{LocationInstanceId}
    Updates political notes for location instance with player and DM sections.

CampaignNotesController (Authorized)

Route Prefix: api/campaigns/{campaignId}/notes

  • GET /
    Retrieves notes for specific entity type and instance. Filters by entityType and instanceId query parameters.
  • POST /
    Creates or updates notes for a campaign entity (location, cast, etc.).

TimeOfDayController (Authorized)

Route Prefix: api/campaigns/{campaignId}/time-of-day

  • GET /
    Retrieves campaign's time of day configuration and current cursor position.
  • PUT /
    Creates or updates time of day with slices. Validates slice durations sum correctly. Broadcasts via SignalR.
  • PATCH /cursor
    Updates time cursor position (0-100%). Broadcasts movement to all connected clients.
  • PATCH /slices/{sliceId}/player-notes
    Updates player-visible notes for a time slice. Broadcasts via SignalR.
  • PATCH /slices/{sliceId}/dm-notes
    Updates DM-only notes for a time slice. Broadcasts via SignalR.

DashboardController (Authorized - DM Only)

Route Prefix: api/dashboard

  • GET /stats
    Retrieves DM dashboard statistics (total campaigns, locations, sublocations, casts, active campaign).
  • GET /import-template
    Downloads a ZIP template for importing library data. Returns compressed file.
  • POST /import
    Imports library data from ZIP file. Processes JSON manifest and embedded images. Returns import results.
  • GET /export
    Exports all DM's library data (casts, locations, sublocations) as ZIP file with JSON and images.

AdminController (Authorized - Admin Only)

Route Prefix: api/admin

  • GET /invite-code
    Retrieves current admin invite code if exists. Returns code and expiration time.
  • POST /invite-code/generate
    Generates a new admin invite code. Invalidates previous code. Sets expiration time.

UsersController (Authorized)

Route Prefix: api/users

  • GET /keywords
    Retrieves list of keywords associated with current user's content (for autocomplete/search).

Service Layers

Core Services (Authentication & Authorization):

  • AuthService - Login, registration, token management, and session handling
  • AuthGuard - Route guard for general authentication checks
  • PlayerGuard - Route guard enforcing Player role requirement
  • DmGuard - Route guard enforcing DM role requirement
  • AdminGuard - Route guard enforcing Admin role requirement
  • CoverGuard - Route guard for cover/landing page access logic
  • AuthInterceptor - HTTP interceptor automatically adding JWT tokens to requests

Hub Services (Real-time Communication):

  • CampaignHubService - SignalR hub for real-time campaign updates (cast visibility changes, time updates, condition assignments, etc.)
📝 Note on Real-time Features: The application uses SignalR for live updates in campaigns. When DMs or players perform certain actions (reveal secrets, assign conditions, update time of day), connected clients in the same campaign receive real-time notifications through the CampaignHub WebSocket connection.
🔐 Note on Role-Based Access: The application supports three user roles: Player, DM, and Admin. Each role has distinct capabilities and visible routes. Players see only campaign and character content. DMs have full library management and campaign control. Admins manage system-level features like invite codes.
